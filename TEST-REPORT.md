# TurnoQuests 1.3.1 — проверка

Версия 1.3.1 исправляет падение всех команд из-за несовместимой сборки 1.3.0 и сохраняет ручное получение награды через красную/зелёную кнопку.

Автоматический набор проверяет:

- ровно 100 квестов и правильный порядок 1–100;
- соответствие каждого квеста своей главе;
- тип, цель, количество, иконку и награду каждого задания;
- выполнение всех 100 заданий правильным событием;
- ожидание ручного клика после выполнения и открытие следующего квеста только после получения награды;
- красное состояние активного квеста и зелёное состояние готовой награды;
- сохранение денежной, бонусной и предметной частей награды после перезапуска;
- блокировку неправильного типа и неправильной цели;
- переход на следующий этап и состояние после №100;
- возврат администратора с №54 на №34;
- сохранение журнала уже выданных наград после сброса;
- офлайн-поиск по UUID и последнему нику;
- сериализацию данных, аудит, историю и ZIP-бэкап;
- ежедневную, еженедельную и общесерверную ротацию;
- отсутствие боссовых заданий, арен и посторонних зависимостей;
- встроенного NPC, Vault и награды ExecutableItems.
- все маршруты игроковых и административных команд;
- ABI-вызовы Paper/Purpur, Vault и Bungee API, включая GUI, чат, снаряды и выдачу EI-предметов;
- запрет выпуска JAR через локальные API-заглушки.

Сборка проверяется в GitHub Actions на Temurin JDK 21, а готовые файлы сохраняются как артефакт workflow.

Итоговое количество проверок и SHA-256 версии 1.3.1 указаны после контрольной сборки Java 21.
