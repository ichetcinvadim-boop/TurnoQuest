# TurnoQuests 1.1.0

Самостоятельная квестовая система для Paper/Purpur 1.21.10: 100 строго последовательных квестов, 10 глав, интерфейс большого сундука, встроенные NPC и боссы.

## Зависимости

- Vault — обязателен для денежных наград (нужен также любой Vault-совместимый провайдер экономики, например EssentialsX Economy).
- ExecutableItems — нужен только для десяти особых предметов в конце глав.

MythicMobs, Citizens, PlaceholderAPI, BetonQuest и другие квестовые/NPC-плагины не нужны. Если старый BetonQuest всё ещё установлен и занимает `/quests`, освободите эту команду или удалите старый плагин.

## Установка

1. Остановите сервер.
2. Положите `TurnoQuests-1.1.0.jar` в `plugins/`.
3. Запустите сервер и выполните `/tq validate`.
4. Встаньте в точку квестового проводника и выполните `/tq npc set`.
5. На каждой арене выполните `/tq boss setarena <ID>`.

## Возможности

- 100 квестов проходят только по порядку; следующий закрыт до выполнения текущего.
- Сброс с №54 на №34 возвращает игрока на №34 и заставляет повторно пройти цепочку дальше.
- Уже выданные награды после сброса не дублируются.
- Денежные награды Vault выдаются даже офлайн; предмет ExecutableItems ждёт входа игрока и свободного слота.
- 54-слотовый GUI, ActionBar-трекер, ежедневные/еженедельные/общие цели, секреты, бонусы без смерти, топ и престиж.
- Встроенный неуязвимый NPC открывает меню по правому клику.
- Десять встроенных боссов на основе ванильных мобов имеют увеличенное здоровье, умения и BossBar.
- Совместное убийство босса засчитывается участникам, нанёсшим минимум 10% общего урона и оставшимся рядом.
- Антинакрутка блоков, Creative/Spectator, мобов из спавнеров и яиц.
- Данные по UUID, аудит административных действий и ZIP-бэкапы.

Предметы глав подобраны в порядке, полезном для дальнейшего пути: лук → куб роста → кирка → топор → скоростные ботинки → сердце бездны → аура брони → тотем → Frostmourne → крылья архангела.

## Команды игрока

- `/quests` — главное меню;
- `/quests 1` … `/quests 10` — выбранная глава;
- `/tq tracker` — переключить трекер;
- `/tq top [страница]` — рейтинг;
- `/tq prestige` — новый круг после всех 100 квестов.

## Административные команды

Команды прогресса работают с офлайн-игроком по UUID или последнему известному нику:

- `/tq info <игрок|UUID>`
- `/tq skip <игрок|UUID> [текущий_квест]`
- `/tq complete <игрок|UUID> [текущий_квест]`
- `/tq reset <игрок|UUID> <1-100>`
- `/tq set <игрок|UUID> <1-100>`
- `/tq progress <игрок|UUID> add|set <число>`
- `/tq reward <игрок|UUID> claim`
- `/tq reward <игрок|UUID> reopen <квест>` — повторная выдача с записью в аудит;
- `/tq history <игрок|UUID> [1-50]`
- `/tq backup`, `/tq validate`, `/tq reload`, `/tq global reset`

Встроенный NPC:

- `/tq npc set` — поставить/перенести NPC в текущую позицию;
- `/tq npc spawn` — восстановить его в сохранённой позиции;
- `/tq npc remove` — удалить.

Встроенные боссы:

- `/tq boss list`
- `/tq boss setarena <ID>` — сохранить арену в текущей позиции;
- `/tq boss spawn <ID> [игрок]`
- `/tq boss removeall`

ID: `ICE_EXECUTIONER`, `HARVEST_LORD`, `MINE_COLOSSUS`, `SAND_TITAN`, `NIGHT_REAPER`, `BLOOD_BERSERKER`, `RIFT_GUARDIAN`, `THUNDER_LORD`, `OMNIRON`, `APOCALYPSE_LORD`.

Игрок также может нажать текущий квест босса в GUI — босс появится на назначенной арене.

## LuckPerms

`turnoquests.use` доступно всем по умолчанию. Полный доступ:

```text
/lp group admin permission set turnoquests.admin true
```

Отдельные права: `turnoquests.admin.info`, `.progress`, `.reset`, `.reward`, `.reload`, `.validate`, `.backup`, `.history`, `.boss`, `.npc`.

## Данные и сборка

Игроки: `plugins/TurnoQuests/data/players/<UUID>.properties`; аудит: `data/audit.log`; бэкапы: `data/backups/`.

Сборка требует JDK 21 и Maven: `mvn package`. В JAR не встраиваются Paper API и Vault API.
