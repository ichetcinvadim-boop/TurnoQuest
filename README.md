# TurnoQuests 1.3.1

Самостоятельная квестовая система для Paper/Purpur 1.21.10: 100 строго последовательных квестов, 10 глав, интерфейс большого сундука и собственный квестовый NPC.

## Зависимости

- Vault — обязателен для денежных наград. Нужен любой Vault-совместимый провайдер экономики, например EssentialsX Economy.
- ExecutableItems — используется только для десяти особых предметов в конце глав.

MythicMobs, Citizens, PlaceholderAPI, BetonQuest, боссы и арены не используются.

## Обновление с предыдущей версии

1. Остановите сервер.
2. Удалите старый JAR и установите `TurnoQuests-1.3.1.jar`.
3. Запустите сервер.

Если в `quests.yml` остались старые задания с боссами, плагин автоматически сохранит их в `quests-before-1.2.0.yml` и установит новую цепочку. Данные и прогресс игроков сохраняются.

## Возможности

- 100 квестов в 10 главах, выполняемых только по порядку.
- 54-слотовый GUI со статусами, прогрессом, описанием и наградами.
- Ручное получение награды: красная кнопка текущего квеста после выполнения становится зелёной.
- Квесты на добычу, строительство, крафт, переплавку, охоту, урон, щит, фермерство, рыбалку, приручение, путешествия, биомы, миры и ванильные достижения.
- Сложность и требования постепенно растут от первых инструментов до финального достижения «Как мы сюда попали?».
- Ежедневные, еженедельные, общесерверные и скрытые задания.
- Бонусы за время и прохождение без смерти.
- ActionBar-трекер, статистика, топ и престиж после №100.
- Встроенный неуязвимый NPC открывает GUI по правому клику.
- Антинакрутка блоков, Creative/Spectator и мобов из спавнеров/яиц.
- Сохранение по UUID, аудит административных действий и ZIP-бэкапы.

## Награды

Каждый квест выдаёт валюту через Vault. Награда не выдаётся автоматически: после выполнения красная кнопка квеста становится зелёной. Игрок нажимает её, получает приз и только после этого открывает следующий квест. В конце каждой главы дополнительно выдаётся полезный предмет ExecutableItems:

1. Астральный лук.
2. Куб роста.
3. Кирка шахтёра.
4. Топор дровосека.
5. Скоростные ботинки.
6. Сердце бездны.
7. Аура прочности брони.
8. Тотем драгонида.
9. Frostmourne.
10. Крылья архангела.

Если Vault недоступен, команда ExecutableItems не сработала или инвентарь заполнен, награда сохраняется, а кнопка остаётся зелёной. Повторный клик продолжит выдачу без дублирования уже полученной части.

## Команды игрока

- `/quests` — открыть главное меню;
- `/quests 1` … `/quests 10` — открыть главу;
- `/tq tracker` — переключить трекер;
- `/tq top [страница]` — рейтинг;
- `/tq prestige` — начать новый круг после 100-го квеста.

## Собственный NPC

Встаньте в нужную точку и выполните:

```text
/tq npc set
```

Дополнительные команды:

- `/tq npc spawn` — восстановить NPC в сохранённой точке;
- `/tq npc remove` — удалить NPC.

NPC создаётся самим TurnoQuests, не получает урон, не двигается и восстанавливается после перезапуска.

В 1.2.1 исправлено создание NPC: чанк загружается заранее, NPC ставится по центру блока, принудительно становится видимым и подсвечивается. После `/tq npc set` в чат выводятся его точные координаты.

## Административные команды

Команды прогресса работают с офлайн-игроком по UUID или последнему известному нику:

- `/tq info <игрок|UUID>`
- `/tq skip <игрок|UUID> [текущий_квест]`
- `/tq complete <игрок|UUID> [текущий_квест]`
- `/tq reset <игрок|UUID> <1-100>`
- `/tq set <игрок|UUID> <1-100>`
- `/tq progress <игрок|UUID> add|set <число>`
- `/tq reward <игрок|UUID> claim`
- `/tq reward <игрок|UUID> reopen <квест>`
- `/tq history <игрок|UUID> [1-50]`
- `/tq backup`, `/tq validate`, `/tq reload`, `/tq global reset`

Сброс с №54 на №34 возвращает игрока на №34 и снова блокирует дальнейшую цепочку. Журнал уже выданных наград сохраняется, поэтому автоматического дублирования нет.

## LuckPerms

`turnoquests.use` доступно всем по умолчанию. Полный доступ администратора:

```text
/lp group admin permission set turnoquests.admin true
```

Отдельные права: `turnoquests.admin.info`, `.progress`, `.reset`, `.reward`, `.reload`, `.validate`, `.backup`, `.history`, `.npc`.

## Файлы

- игроки: `plugins/TurnoQuests/data/players/<UUID>.properties`;
- аудит: `plugins/TurnoQuests/data/audit.log`;
- бэкапы: `plugins/TurnoQuests/data/backups/`;
- цепочка: `plugins/TurnoQuests/quests.yml`.

Сборка требует JDK 21 и Maven. В итоговый JAR не встраиваются Paper API и Vault API.

## Исправление 1.3.1

Версия 1.3.0 была собрана через устаревшие Bukkit-заглушки и содержала пять несовместимых вызовов API. Из-за этого сервер показывал «при выполнении команды произошла непредвиденная ошибка». Версия 1.3.1 компилируется только против настоящего Paper API и автоматически проверяет все обращения к Paper, Vault и Bungee API перед выпуском. Командные и стартовые ошибки теперь полностью записываются в консоль с названием исключения.
